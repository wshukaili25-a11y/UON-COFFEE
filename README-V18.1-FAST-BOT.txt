UON Hub V18.1 — Fast Telegram Bot

سبب التأخير الذي تم إصلاحه:
- كان البوت ينتظر قاعدة البيانات قبل إيقاف علامة التحميل في Telegram.
- تحديث last_seen كان يوقف كل ضغطة.
- سجل العمليات كان يُكتب قبل إظهار النتيجة.
- عدادات الطلبات كانت تُحسب بالتتابع جدولًا بعد جدول.
- إشعارات الطلبات القادمة من الموقع كانت تؤخر استجابة web-submit.

التعديلات:
- إيقاف علامة تحميل زر Telegram فورًا قبل أي استعلام.
- Cache لصلاحيات المشرف لمدة 60 ثانية.
- تحديث last_seen في الخلفية.
- سجل العمليات في الخلفية.
- حساب عدادات الطلبات بالتوازي.
- إشعارات web-submit في الخلفية.
- إضافة /ping لاختبار سرعة استجابة البوت.
- لا يوجد SQL جديد.
- لم يتم تعديل تصميم الموقع أو وظائفه.

النشر:
supabase functions deploy telegram-admin --no-verify-jwt

بعد النشر:
1. أرسل /ping — المفروض يرد بسرعة.
2. أرسل /start.
3. جرّب الخدمات، الصيانة، والطلبات المعلقة.
