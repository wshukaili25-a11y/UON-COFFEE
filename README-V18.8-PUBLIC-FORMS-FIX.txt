UON Hub V18.8 — Public Forms & RLS Hotfix

المشكلة الأساسية:
كانت نماذج الطلاب تستخدم Prefer: return=representation بعد INSERT.
الجداول تسمح للزائر بإضافة سجل pending، لكنها لا تسمح له بقراءة السجل pending.
لذلك قد ينجح الإدخال ثم يفشل طلب إعادة السجل برسالة تبدو كأنها RLS.

الإصلاحات:
1) إضافة submitPending() التي تنشئ UUID محليًا وترسل INSERT مع return=minimal.
2) إزالة Authorization: Bearer من المفتاح publishable، والإبقاء على apikey فقط.
3) تحديث النماذج:
   - الاقتراحات
   - الملخصات
   - مجموعات واتساب
   - التقييمات
   - المشاريع الطلابية
   - الاعترافات
   - بلاغات الروابط
4) التحقق من الحقول الإلزامية في نموذج الاقتراحات.
5) إرسال إشعار تيليجرام باستخدام ID الصحيح للطلب.
6) جعل تتبع الاستخدام والأخطاء يستخدم return=minimal.
7) تحديث cache version إلى 18.8.

خطوات النشر:
1. نفّذ UON_HUB_V18_8_PUBLIC_FORMS_FIX.sql داخل Supabase SQL Editor.
2. ارفع ملفات الموقع إلى GitHub/Vercel.
3. لا تحتاج لنشر Edge Function لهذا الإصلاح.
4. افتح الموقع في نافذة خاصة أو امسح الكاش وجرب كل نموذج.

اختبارات سريعة:
- إرسال اقتراح.
- رفع ملخص.
- إرسال رابط مجموعة.
- إرسال تقييم.
- إرسال مشروع.
- إرسال اعتراف.
- إرسال بلاغ عن رابط.
ثم تأكد أن الطلب يظهر Pending في لوحة الإدارة والبوت.
